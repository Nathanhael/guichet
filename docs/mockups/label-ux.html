T4 — Hybrid (recommended)

Combines T1 + T3. Auto-hide empty rows for end-users, per-ticket collapse for support. Best of both: zero chrome when not needed, quick access when it is.

Rules for T4 (recommended):
• End-user (agent view): row never shown — they don't manage labels
• Support/admin, ticket has labels: row expanded by default, chevron collapses to compact strip (T1 style)
• Support/admin, ticket empty: row shown expanded with `+ LABEL` button only
• Collapse state persisted per-ticket in localStorage (key: `guichet.labelRow.collapsed.${ticketId}`)
• Auto-hide in focus/compact mode (inherited from existing header rules)

No global setting needed — the toggle lives on the row itself and is cheaper to discover than a SettingsPopover entry.
