# Admin Dashboard Rework — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the admin dashboard rework described in `docs/superpowers/specs/2026-04-20-admin-dashboard-rework-design.md` — deleting Time-in-Status, Team Satisfaction panel, and Agent Performance chart; adding Demand Heatmap, Users Needing Attention, Avg Resolution KPI, and upgrading Support Workload to a multi-metric table.

**Architecture:** Dashboard redesign is frontend-heavy. Backend changes are additive to `stats.getGlobalStats` (heatmap aggregation, per-support resolution minutes, dept-avg for users) plus straight deletions (`rating.getStaffRatings`, `statusRouter.getAgentStats`/`getTeamStats`, `statusTracking` service, two status tables, socket hooks). No new tables, one drop migration. All polling stays at 30s.

**Tech Stack:** Drizzle ORM + PostgreSQL 18, tRPC 11 + Zod, Recharts 3, React 19, Vitest + Playwright.

**Open questions from spec (must be resolved before Phase 2 starts):** Q1–Q5 in spec §Open Questions.

## Phase 3 — Backend stats upgrade

### Task 7: Add resolution-time aggregation

**Files:** `server/trpc/routers/stats.ts`

- [ ] **Step 7.1** — Add aggregation for `avgResolutionMinutes` at the top-level: for tickets with `status = 'closed'` and a `closedAt`, compute `AVG(EXTRACT(EPOCH FROM (closed_at - created_at)))/60` over the filtered set. Return as integer minutes.
- [ ] **Step 7.2** — Add per-support `avgResolutionMinutes` to the `supportStats` map computation — restrict to tickets where that support was the resolver (`supportId = user AND status = 'closed'`).
- [ ] **Step 7.3** — Add per-support `avgResponseMinutes` to `supportStats` (individual breakdown of the existing global response time). Compute using the same `first_staff_response_at - created_at` math, filtered by `supportId`.
- [ ] **Step 7.4** — Update the return object's type inference / Zod shape if one exists.

**Verify:** Run `docker compose exec server npm test -- stats` — add a new test asserting the return keys include `avgResolutionMinutes`, each `supportStats[i]` has `avgResolutionMinutes` + `avgResponseMinutes`.

### Task 8: Add Demand Heatmap aggregation

**Files:** `server/trpc/routers/stats.ts`

- [ ] **Step 8.1** — Add a separate query that groups filtered tickets by `EXTRACT(DOW FROM created_at AT TIME ZONE $tz)` and `EXTRACT(HOUR FROM created_at AT TIME ZONE $tz)`, returning `{ dow, hour, count }` rows. Timezone comes from `partner.businessHoursSchedule.timezone` (fall back to `'UTC'` if unset).
- [ ] **Step 8.2** — Return the rows as `demandHeatmap` array in the stats payload. Do **not** fill in zero-count cells — client handles that.

**Verify:** Add a stats test: seed 3 tickets at known timestamps, assert `demandHeatmap` has 3 rows with correct `dow` / `hour` / `count`.

### Task 9: Add Users Needing Attention aggregation

**Files:** `server/trpc/routers/stats.ts`

- [ ] **Step 9.1** — Drop the existing `agentStats` builder block (lines ~406–419). Remove unused `AgentMapEntry` type.
- [ ] **Step 9.2** — Add the new `userAttention` aggregation:
    - Group live tickets by `agentId` to get per-user `total`.
    - Join on `memberships.departments` to derive the user's primary department (first element; null if generalist).
    - Group by `deptId` to get `deptTotal` + `deptUserCount`; `deptAvg = deptTotal / deptUserCount`.
    - Compute `deltaPct = (total - deptAvg) / deptAvg * 100` per row.
    - **Filter**: drop rows where `total < 3` (per spec Q3 default).
    - **Null-safe**: when `deptAvg === 0`, emit `deltaPct = null`.
    - Sort by `deltaPct DESC NULLS LAST`, limit top 20.
- [ ] **Step 9.3** — Return as `userAttention` in the stats payload.

**Verify:** Add a stats test: seed a dept with one outlier user (10 tickets) and two baseline users (1 ticket each). Assert the outlier surfaces with `deltaPct ≈ +400%`.

**Commit after Tasks 7–9**: `feat(dashboard): add resolution/heatmap/user-attention aggregations to getGlobalStats`
